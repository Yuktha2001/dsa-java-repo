1. Big-O Cheat Sheet (Important for Interviews)
Complexity	Name	Example
O(1)	Constant	Access array element
O(log n)	Logarithmic	Binary Search
O(n)	Linear	Linear Search
O(n log n)	Linearithmic	Merge Sort
O(n²)	Quadratic	Nested loops
O(2ⁿ)	Exponential	Recursive Fibonacci
O(n!)	Factorial	Traveling Salesman
Visual Growth of Time Complexity
n	O(1)	O(log n)	O(n)	O(n²)
10	1	3	10	100
100	1	7	100	10000
1000	1	10	1000	1000000

You can see quadratic grows very fast, which is why we avoid O(n²) if possible.

2. How to Identify Time Complexity (Simple Rules)
Rule 1: Ignore Constants
O(2n) → O(n)
O(5n + 3) → O(n)
Rule 2: Drop Lower Terms
O(n² + n + 5) → O(n²)

Because n² dominates.

Rule 3: Loops Multiply Complexity
for(i=0;i<n;i++)
   for(j=0;j<n;j++)

Complexity

n × n = O(n²)
Rule 4: Nested loops = Multiply
for(i=0;i<n;i++)
   for(j=0;j<i;j++)

Still O(n²).

Rule 5: Logarithmic Pattern

If input divides repeatedly

n → n/2 → n/4 → n/8

Complexity = O(log n)

Example:

Binary Search.

3. Practice Questions (Very Important)

Try to guess the complexity before seeing answer.

Question 1
for(int i=0;i<n;i++)
{
   System.out.println(i);
}

Answer:

O(n)

Because loop runs n times.

Question 2
for(int i=0;i<n;i++)
{
   for(int j=0;j<n;j++)
   {
      System.out.println(i+j);
   }
}

Answer:

O(n²)

Because nested loops.

Question 3
for(int i=0;i<n;i++)
{
   for(int j=0;j<100;j++)
   {
      System.out.println(i+j);
   }
}

Answer:

O(n)

Because 100 is constant.

Question 4
for(int i=1;i<n;i=i*2)
{
   System.out.println(i);
}

Values of i:

1
2
4
8
16
...

Answer:

O(log n)
Question 5
for(int i=n;i>0;i=i/2)
{
   System.out.println(i);
}

Answer:

O(log n)

5. Most Important Complexities to Remember

Interviewers mostly expect you to know these:

Algorithm	Complexity
Linear Search	O(n)
Binary Search	O(log n)
Bubble Sort	O(n²)
Merge Sort	O(n log n)
Quick Sort	O(n log n) average
